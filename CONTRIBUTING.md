# Contributing

Thanks for your interest in improving this project! A few guidelines to make collaboration smooth.

## How to contribute
1. Fork the repository.
2. Create a feature branch:
   ```
   git checkout -b feat/your-feature
   ```
3. Make changes in a clear, atomic commits.
4. Run tests (if any) and ensure linting passes.
5. Open a Pull Request describing the change.

## Coding standards
- Python: follow PEP8.
- JavaScript: modern ES6+ patterns.
- Keep UI changes minimal and accessible.

## Issues & Feature Requests
- Open an issue with a clear title and steps to reproduce.
- Tag maintainers if the issue is urgent.

## Security
If you discover a security vulnerability (e.g., API key leakage), please contact the maintainer privately instead of opening a public issue.
